Semana 03

Introducción al CSS

En esta semana del curso, nos introdujimos a CSS, que es lo que se usa para darle estilo a HTML.

¿Qué es CSS?

CSS es una forma de definir el formato del documento HTML, su nombre viene de Cascading Style Sheets, lo que significa “Hojas de Estilo en Cascada”.

El concepto de "Cascada" en CSS describe cómo se combinan y aplican las reglas de estilo de manera jerárquica, teniendo en cuenta la especificidad, el orden de aplicación y la herencia para determinar el estilo final de cada elemento en una página web.

Una de las ventajas de usar las hojas de estilo es que permiten hacer un diseño consistente y fácil de modificar.

Sintaxis de las hojas de estilo:

Una hoja de estilo es un fichero de texto plano (sin formato) en el que se define el aspecto de las etiquetas de una página web:

body {
color: black;
background-color: white;
}
h1 {
color: white;
background-color: black;
font-family: 'Franklin Gothic Medium', 'Arial Narrow', Arial, sans-serif;
}

Las hojas de estilo están formadas por varias sentencias, estas están formadas por un selector y un bloque de declaración. El bloque de declaración empieza y acaba con llaves {}.

Cada declaración está compuesta por una o varias propiedades y sus valores correspondientes. Estas se separan entre sí por puntos y comas (;).

El valor (o valores) van separados de las propiedades por dos puntos (:).

En las hojas de estilo, los selectores son la parte de las reglas que indican al navegador a qué elementos se van a aplicar las propiedades incluídas en las declaraciones.

Entre ellos, se encuentran:

  • Selector universal: *
  • Selector de tipo: E
  • Selector de clase: E.e
  • Selector de identificador (id): E#e
  • Selector de descendientes: E F
  • Selector de hijos: E > F
  • Selector de consecutivos: E + F
  • Selector de grupo: E, F

Reglas de aplicación de los estilos:

Si se define la misma propiedad para la misma etiqueta con el mismo selector en dos sitios distintos, las reglas de precedencia son las siguiente:

  1. Las propiedades definidas en un atributo style se imponen a las propiedades definidas en la etiqueta style, a las propiedades definidas en una hoja de estilo enlazada y a las propiedades por defecto del navegador.
  2. Las propiedades definidas en la etiqueta style se imponen a las propiedades definidas en una hoja de estilo enlazada y a las propiedades por defecto del navegador.
  3. Las propiedades definidas en una hoja de estilo enlazada se imponen a las propiedades por defecto del navegador.
  4. En caso de no aplicar ningún estilo en particular a un elemento, el navegador lo mostrará según sus propiedades por defecto (por ejemplo, los títulos de primer nivel h1 se suelen mostrar en negrita y al doble del tamaño del resto del texto).

Si las propiedades en conflicto se encuentran definidas en diferentes hojas de estilo, el navegador aplica el valor definido en la última hoja de estilo enlazada (es decir, en el último enlace link del head).
