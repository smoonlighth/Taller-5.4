Semana 01

Introducción al Desarrollo Web


El desarrollo web es el proceso de creación y mantenimiento de los sitios web a través de diferentes lenguajes de programación. Cubre funciones como el registro, los sistemas de gestión de contenido y cualquier aplicación de base de datos. También implica el diseño de la interfaz (frontend) y la lógica del servidor (backend).


Roles en el Desarrollo Web

Frontend (Cliente): Es la parte que interactúa con el usuario, tanto en imagen como en función. Por ello está íntimamente relacionada con la experiencia del usuario (UX) y la interfaz de usuario (IU).

Backend: Maneja la lógica del servidor, la base de datos, y la integración de la API. Asegura la seguridad, la autenticación, la autorización, y la escalabilidad de la aplicación.


Tecnologías utilizadas

HTML (HyperText Markup Language): Lenguaje de marcado utilizado para estructurar y presentar el contenido en la web.

CSS (Cascading Style Sheets): Lenguaje de hojas de estilo utilizado para describir la presentación de un documento HTML.

JavaScript: Lenguaje de programación que permite crear contenido dinámico e interactivo en la web.

Backend(Servidor): Se refiere a la parte que está en contacto directo con el servidor; es donde se aplica el código de programación para crear la estructura. Permanece en un segundo plano a cargo de la accesibilidad, actualización, bases de datos y cambios del sitio.


Lenguajes de programación

PHP, Python, Ruby, Java, Node.js.


Bases de datos

MySQL, PostgreSQL, MongoDB.


Etapas del desarrollo de Software

Planificación: documentación de los requerimientos del proyecto y tecnologías a utilizar. Se definen plazos de las etapas. Diseño: definición de flujos e interacción del usuario y diseño de la interfaz gráfica.

Desarrollo: construcción de las bases de datos y programación de la lógica a nivel cliente/servidor. Codificación de la interfaz y funcionalidad.

Despliegue: se pasa a producción y se coloca a disposición de los usuarios, publicación del sitio en un servidor web. Soporte: corrección de errores, pequeños cambios y mantenimiento a la base de datos.




Semana 02

¿Qué es HTML?


HTML (HyperText Markup Language) es el lenguaje de marcado estándar para crear páginas web. Define la estructura y contenido de los elementos de la página mediante el uso de etiquetas y atributos.


Conceptos básicos

-Etiquetas: palabras clave encerradas entre corchetes angulares (“elemento”) que indican al navegador cómo mostrar el contenido.
-Elementos: se encuentran entre una etiqueta de apertura y una de cierre se considera un elemento.
-Atributos: características adicionales que se añaden a las etiquetas para proporcionar más información. Ejemplos de uso común: src, class, href, id, style.


Estructura básica de una página HTML

  1. <!DOCTYPE html>: Declara el tipo de documento HTML.
  2. <html></html>: Raíz de la página.
  3. <head></head>: Contiene metadatos como el título.
  4. <body></body>: Contiene el contenido visible de la página.

Semana 03

Introducción al CSS


En esta semana del curso, nos introdujimos a CSS, que es la herramienta que se usa para darle estilo a un documento HTML.


¿Qué es CSS?

CSS es una forma de definir el formato del documento HTML, su nombre viene de Cascading Style Sheets, lo que significa “Hojas de Estilo en Cascada”. El concepto de "Cascada" en CSS describe cómo se combinan y aplican las reglas de estilo de manera jerárquica, teniendo en cuenta la especificidad, el orden de aplicación y la herencia para determinar el estilo final de cada elemento en una página web.

Una de las ventajas de usar las hojas de estilo es que permiten hacer un diseño consistente y fácil de modificar.


Sintaxis de las hojas de estilo

Una hoja de estilo es un fichero de texto plano (sin formato) en el que se define el aspecto de las etiquetas de una página web:


body { 

            color: black; 

            background-color: white; 

        }
        

h1 { 

            color: white; 

            background-color: black; 

            font-family: 'Franklin Gothic Medium', 'Arial Narrow', Arial, sans-serif; 

        }


Las hojas de estilo están formadas por varias sentencias, estas están formadas por un selector y un bloque de declaración. El bloque de declaración empieza y acaba con llaves {}.

Cada declaración está compuesta por una o varias propiedades y sus valores correspondientes. Estas se separan entre sí por puntos y comas (;).

El valor (o valores) van separados de las propiedades por dos puntos (:).

En las hojas de estilo, los selectores son la parte de las reglas que indican al navegador a qué elementos se van a aplicar las propiedades incluídas en las declaraciones.


Entre ellos, se encuentran:


  • Selector universal: *
  • Selector de tipo: E
  • Selector de clase: E.e
  • Selector de identificador (id): E#e
  • Selector de descendientes: E F
  • Selector de hijos: E > F
  • Selector de consecutivos: E + F
  • Selector de grupo: E, F



    • Reglas de aplicación de los estilos:


    Si se define la misma propiedad para la misma etiqueta con el mismo selector en dos sitios distintos, las reglas de precedencia son las siguiente:


    1. Las propiedades definidas en un atributo style se imponen a las propiedades definidas en la etiqueta style, a las propiedades definidas en una hoja de estilo enlazada y a las propiedades por defecto del navegador.

    2. Las propiedades definidas en la etiqueta style se imponen a las propiedades definidas en una hoja de estilo enlazada y a las propiedades por defecto del navegador.

    3. Las propiedades definidas en una hoja de estilo enlazada se imponen a las propiedades por defecto del navegador.

    4. En caso de no aplicar ningún estilo en particular a un elemento, el navegador lo mostrará según sus propiedades por defecto (por ejemplo, los títulos de primer nivel h1 se suelen mostrar en negrita y al doble del tamaño del resto del texto).

    Si las propiedades en conflicto se encuentran definidas en diferentes hojas de estilo, el navegador aplica el valor definido en la última hoja de estilo enlazada (es decir, en el último enlace link del head).




    Semana 04

    Introducción al JavaScript


    ¿Qué es JavaScript?


    Es un lenguaje de programación que los desarrolladores utilizan para hacer páginas web interactivas.
    Para incluir un archivo js externo en una página html, utilizamos la etiqueta <script> de la siguiente manera:
    <script src="js/script.js"></script>
    O podemos incluirlo directamente dentro de la misma etiqueta, aunque por cuestiones de organización no es recomendable.


    Funciones

    Las funciones son el elemento principal para programar nuestro programa en "bloques". Para declarar una función, usaremos la palabra clave function seguido del nombre de la misma, sus parámetros y luego entre llaves el cuerpo de la función.



      function mostrarMensaje() {
    
            alert( 'Hola a todos!' );
    
        }

    Para invocarla, usaremos su nombre seguido de paréntesis curvos. mostrarMensaje();


    Las funciones pueden recibir más de un parámetro, estos son datos que se entregaran a la función en el momento de su declaración, y pueden ser datos de cualquier tipo, o incluso otra función.
    En el ejemplo, la función recibe 2 parámetros: from y text:



     function showMessage(from, text) { // arguments: from, text
    
        alert(from + ': ' + text);
    
        }

    
        showMessage('Ann', 'Hello!'); // Ann: Hello!

    
        showMessage('Ann', "What's up?"); // Ann: What's up?

    
        Una función puede devolver valores como resultado a la línea de código que invocó la función.

    
        function sum(a, b) {
    
        return a + b;}
    
        let result = sum(1, 2);
    
        alert( result ); // 3

    ¿Qué es una variable?


    Es un espacio en memoria que almacena un dato. Se declaran con const (para constantes), y let (para variables). También se puede ver var, aunque ya no se utiliza.


    Variables globales


    Las variables globales o externas son declaradas fuera del ámbito del bloque, y pueden ser accedidas tanto fuera como dentro de los bloques, por ejemplo:

     let userName = 'John';

    
        
        function showMessage() {
    
          let message = 'Hello, ' + userName;
    
          alert(message);
    
        }

    
        showMessage(); // Hello, John

    
        alert(userName); // John




    Document Object Model (DOM)


    Permite a los programadores web acceder y manipular los elementos de las páginas HTML a través de objetos llamados nodos. Estos están interconectados y representan los contenidos de las páginas web y las relaciones entre ellos. Por su aspecto, la unión de todos los nodos se denomina "árbol de nodos ".
    En todas las páginas web, la raíz principal del árbol es el objeto document.


    Tipos de nodo


    En en las páginas HTML se manipulan habitualmente cuatro tipos de nodos:


  • Document: nodo raíz del que derivan todos los demás nodos del árbol.
  • Element: representa cada una de las etiquetas HTML. Se trata del único nodo que puede contener atributos y el único del que pueden derivar otros nodos.
  • Attr: se define un nodo de es te tipo para representar cada uno de los atributos de las etiquetas HTML, es decir, uno por cada par atributo=valor.
  • Text: nodo que contiene el texto encerrado por una etiqueta HTML.


    • Acceso a nodos

    Es posible acceder de forma sencilla a todos los atributos HTML y todas las propiedades CSS de cualquier elemento de la página. Los atributos HTML de los elementos de la página se transforman automáticamente en propiedades de los nodos.

    Para acceder a su valor, simplemente se indica el nombre del atributo HTML detrás del nombre del nodo,


    wikipedia
     let enlaces = document.getElementsByTagName('a')
    
        alert(enlaces[0].href);


    Existen varias formas de seleccionar etiquetas:


    document.getElementsByTagName(‘element’): devuelve un array (arreglo) con todos los elementos(element) de la página.
    getElementsByName('nameHTML'): devuelve un array con todos los elementos con determinado nombre.
    getElementById('id'): devuelve un solo elemento HTML con el id especificado.
    getElementsByClassName('class'): devuelve un array con todos los elementos de la clase indicada.


    Para acceder a los estilos de los elementos, utilizamos la propiedad style.

    document.getElementById("id").style.backgroundColor // Correcto
    document.getElementById("id").style.background-color // Incorrecto

    Nota: Si el nombre de una propiedad CSS es compuesto, se accede a su valor eliminando todos los guiones medios (-) y escribiendo en mayúscula la letra siguiente a cada guión medio: Como la palabra class está reservada por JavaScript, no es posible utilizarla para acceder al atributo class del elemento HTML. En su lugar, DOM utiliza el nombre className para acceder al atributo class de HTML.



    Creación de un nodo de tipo Element que representa al elemento


    let myElement = document.createElement('h1');


    Creación de un nodo de tipo Text que representa el contenido del elemento


    let myText = document.createTextNode('El título principal');

    Adición del nodo Text como hijo del nodo Element:


    myElement.appendChild(myText);


    Añadir el nodo Element a la página, como hijo del elemento deseado (en este ejemplo, el <body>):

    document.body.appendChild(myElement);



    Eliminar un nodo es más sencillo:


    Obtención del nodo:

    let myElement = document.getElementById(id);

    Eliminación del nodo:

    myElement.remove();






    Clases


    Los elementos del DOM tienen una propiedad llamada classList, un array que contiene todas las clases que tiene el elemento. Además, tiene varios métodos adicionales:


    add: Añade una clase.
    remove: Elimina una clase.
    toggle: Añade una clase si no está presente y si lo está, la elimina.
    contains: Revisa si la clase está presente en el elemento y retorna true o false.


    ¿Qué es JSON?


    JSON (JavaScript Object Notation) es un formato de datos basado en texto que sigue la sintaxis de objeto de JavaScript. Son cadenas de texto útiles cuando se quiere transmitir datos a través de una red, y deben ser convertidos a objetos nativos de JavaScript cuando se requiera acceder a sus datos. JavaScript posee un objeto global JSON que tiene los métodos disponibles para convertir entre ellos. Es posible incluir los mismos tipos de datos básicos dentro de un JSON que en un objeto estándar de JavaScript - cadenas, números, arreglos, booleanos, y otros literales de objeto.


    ¿Qué es HTTP?


    HTTP (HyperText Transfer Protocol) funciona como un protocolo de pedido-respuesta entre el cliente y el servidor. Un navegador web puede ser el cliente, y la aplicación en una computadora que aloja un sitio web puede ser el servidor. HTTP define un conjunto de métodos de petición para indicar la acción que se desea realizar para un recurso determinado. Cada uno de ellos implementan una semántica diferente: Método GET: es el método por defecto. Su función es solicitar un recurso específico, por lo tanto las peticiones que utilizan este método sólo deben utilizarse para obtener información. Método POST: se utiliza para enviar una entidad a un recurso en específico al servidor, causando a menudo un cambio en el estado o efectos secundarios del mismo. En este caso los parámetros se envían en el cuerpo del mensaje, y no en la URL misma
    (cómo sí lo hace el método GET):



    Fetch


    Permite realizar pedidos a servidores de forma asincrónica, es decir, sin tener que refrescar la página.


    La sintaxis básica es:

    let promise = fetch(url, [options])
    url: la URL a la que se envía el pedido
    options: parámetros opcionales: método HTTP, encabezados, etc.

    Cuando se realiza una solicitud GET sin especificar opciones, fetch devuelve una promesa que se resuelve con un objeto Response cuando el servidor responde. En esta fase, se puede consultar el estado HTTP para verificar si la respuesta fue exitosa. La promesa se rechaza en caso de problemas como fallos de red o URL inválida.

    El objeto Response tiene propiedades como status (código HTTP) y ok (booleano que indica si el código está entre 200-299). Para manejar la promesa de manera más sencilla, se puede usar await dentro de una función async, que pausa la ejecución hasta que la promesa se resuelve o se rechaza.


    Además, Response ofrece métodos para leer el cuerpo de la respuesta en diferentes formatos, como response.text() para texto y response.json() para JSON.
